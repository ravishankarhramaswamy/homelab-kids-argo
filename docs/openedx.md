# Open edX workflow

Open edX is managed as rendered manifests stored in Git. We do not run Tutor inside Argo CD.

## Render flow
1. Install and configure Tutor locally.
2. Run `scripts/render-openedx.sh` to render manifests into `applications/openedx/rendered/`.
3. Review and commit the rendered output.
4. Argo CD syncs the Kustomize target under `applications/openedx/`.

## Hostnames
- LMS: `openedx.family.home.arpa`
- CMS (Studio): `studio.openedx.family.home.arpa`

Override with `OPENEDX_LMS_HOST` and `OPENEDX_CMS_HOST` when rendering.
The script also accepts `GATEWAY_NAME`, `GATEWAY_NAMESPACE`, and `GATEWAY_SECTION` for Envoy Gateway routing.

## Secrets
- Secrets are intentionally removed from rendered output.
- Create the required secrets out of band based on your Tutor configuration.
- See `shared/secrets/examples/openedx-secrets.example.yaml` for a placeholder scaffold.

## Patches
- Store Kustomize patches in `applications/openedx/patches/`.
- Wire them into `applications/openedx/kustomization.yaml` as needed.
- HTTPRoutes are generated by `scripts/render-openedx.sh` when service names are detected.
